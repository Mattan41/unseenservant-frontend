## Feature structure
Each feature follows the same pattern:
- `FeatureService.js` вЂ” all API calls for the feature
- `featureStore.js` вЂ” Pinia store (state + actions)
- `components/` вЂ” UI components used within the feature
- `views/` вЂ” routable pages for the feature

## API Architechture & Communication
```
Internal:  axios.js             в†’ apiClient.js    в†’ FeatureService.js в†’ store
External:  open5e-axios.js      в†’ open5eClient.js в†’ FeatureService.js в†’ store
```
- `api/lib/axios.js`: Axios instance. Adds JWT header to requests, handles 401 (clears auth, shows notification).
- `api/apiClient.js`: Wrapper around axios. Blocks write operations (POST/PUT/PATCH/DELETE) if user is in guest mode. All services import from here вЂ” never directly from axios. Base URL configured in here.
- External APIs (e.g. Open5e) will be called directly from the browser using a separate Axios instance or fetch вЂ” not proxied through the backend

## Authentication
- JWT-based auth with refresh handled in `authStore`
- OAuth2 via Google/GitHub вЂ” redirect handled by `OAuthRedirect.vue`
- Auth state persisted in `authStore` (Pinia)

## Guest mode
`apiClient.js` has pre-built logic for guest mode via `authStore.isGuest`. GET requests pass through, write operations return `null` silently. Guest mode is not fully implemented yet вЂ” stores and components need to handle `null` responses.

## Component conventions (target pattern)
- **Smart components** (views, containers): handle store calls and logic
- **Dumb/presentational components**: receive props, emit events, no store access
- **Base components**: generic UI building blocks in `src/components/base/`
- Always Composition API with `<script setup>` вЂ” no Options API components

### State management
- Pinia stores are the single source of truth
- Components read state via `storeToRefs()` for reactivity
- API calls live in Service files, called from store actions

### Routing
- Routes requiring login use `meta: { requiresAuth: true }`
- Auth guard in `router/index.js` handles redirect to `/login`
- `authStore.initializeAuth()` is called once on first navigation

## Other
- Always import with `@/` alias, never relative `../` paths
